### 4. Word/Google Docs Parsing Compatibility

**Location in Instructions:** After `<bullet_display_and_grouping_rules>`

```xml
<word_google_docs_parsing_compatibility>
  <priority>CRITICAL</priority>
  <version>6.4.1-update-1</version>
  <date>January 2026</date>
  
  <purpose>
    Handle resumes copied from Word/Google Docs where bullet formatting 
    and indentation is lost in plain text conversion.
  </purpose>

  <problem_addressed>
    When copying from Word/Google Docs, special bullet characters (â€¢) and 
    indentation formatting are lost. This causes the parser to group all 
    content under the first position header.
    
    Solution: Implement intelligent position header detection based on patterns
    rather than relying on indentation or bullet markers.
  </problem_addressed>

  <position_header_detection_algorithm>
    A line is identified as a position header if it:
    
    1. Contains position markers: " at " OR "|" OR company keywords
    2. Has reasonable length: 10-150 characters
    3. Is NOT a bullet marker (â€¢, -, *, o, â—¦, â—‹, numbered)
    
    Position markers include:
    - " at " pattern (Job Title at Company Name)
    - "|" separator (Title | Company | Dates)
    - Company keywords: Inc, LLC, Corp, Company, Corp., Ltd, etc.
    - Date patterns: YYYY, Month Year, dates
    
    When a position header is detected:
    - Save previous position (if it has collected bullets)
    - Start new position with this line as header
    - Collect following lines as bullets until next header found
  </position_header_detection_algorithm>

  <bullet_marker_support>
    The parser supports bullets from any source:
    
    Standard markers:
    - â€¢ (standard bullet)
    - - (hyphen)
    - * (asterisk)
    
    Unicode bullets:
    - o â—¦ â—‹ â— â–  â–¡ â–ª
    
    Numbered lists:
    - 1. 2. 3. etc.
    
    Word/Google Docs:
    - Plain text with intelligent position header detection
    - Lines between position headers are treated as bullets
  </bullet_marker_support>

  <implementation_notes>
    - Once a position header is detected, all following non-header lines become bullets
    - Bullets shorter than 5 characters are skipped (noise reduction)
    - Remove all bullet markers and leading whitespace from collected text
    - If a line looks like a position header, save current position and start new one
  </implementation_notes>
</word_google_docs_parsing_compatibility>
```
