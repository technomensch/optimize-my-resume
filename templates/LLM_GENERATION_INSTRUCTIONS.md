## üîí Schema Enforcement Rules

### Rule 1: Tag Names - EXACT MATCH REQUIRED

| ‚úÖ CORRECT | ‚ùå WRONG |
|-----------|---------|
| `<professional_summary>` | `<summary>` |
| `<key_achievements>` | `<achievements>` |
| `<hard_skills_demonstrated>` | `<technical_skills>` |
| `<impact_metrics>` | `<metrics>` |
| `<contract_type>` | `<employment_type>` |

### Rule 2: Section Order - MUST FOLLOW TEMPLATE

Sections MUST appear in this order within `<position>`:
1. metadata
2. professional_summary
3. core_responsibilities
4. key_achievements
5. hard_skills_demonstrated
6. soft_skills_demonstrated
7. tools_technologies
8. impact_metrics
9. industry_domain
10. methodology
11. strategic_decisions
12. team_scope
13. honest_limitations

### Rule 3: Date Formats - STANDARDIZED

**Dates:**
- Format: `Month Year` (e.g., "January 2025")
- Current: `Present`
- Unknown: `Not specified`

**Duration:**
- Format: `N months` or `N years` or `N months (description)`
- Example: `15 months` or `2 years` or `6 months (temporary assignment)`

### Rule 4: Achievement Structure - USE CONTEXT/ACTION/RESULT/IMPACT

For structured achievements (preferred):

```xml
<achievement id="1" metric_type="deployment">
  <context>What was the situation?</context>
  <action>What did you do?</action>
  <result>What was the measurable outcome?</result>
  <impact>What was the broader significance?</impact>
</achievement>
```

OR simple bullets (acceptable):
```xml
<key_achievements>
  - Achievement 1 with metrics
  - Achievement 2 with metrics
</key_achievements>
```

## üé® Style & Tone Guidelines

### Professional Summary
- **Length:** 2-4 sentences
- **Include:** Role descriptor + key metrics + major responsibilities + impact
- **Tone:** Professional, achievement-focused
- **Example:** "Agile Business Analyst who resolved a 6-month project delay over 8 months by authoring 700+ user stories with 98% definition-of-done compliance..."

### Metrics & Numbers
- **Always quantify** where possible
- **Use specific numbers:** "80%" not "significantly"
- **Include units:** "4 hours ‚Üí 2 hours" not "halved time"
- **Show before/after:** "0% to 100% coverage"

### Bullet Points
- **Start with action verbs** (past tense for completed work)
- **Include metrics** wherever possible
- **Be specific:** "Created 150+ test cases" not "Created test cases"
- **Show impact:** Include "achieving X" or "resulting in Y"

## üîÑ Generation Workflow

### For New Job History (from bullets):

```
1. Review bullet points provided by user
2. Load template: templates/job_history_template.xml
3. For each position:
   a. Extract key information from bullets
   b. Populate ALL sections (use "Not applicable" if no data)
   c. Follow exact tag names and order from template
   d. Include metrics and quantification
4. Generate version history entry
5. Add file header
6. Validate structure matches template
7. Deliver both .txt (XML) and .md (Markdown) formats
```

### For Updates (adding to existing):

```
1. Read existing job history file
2. Identify version number and increment
3. Load template for reference
4. Add new position(s) following exact schema
5. Update version history with changes
6. Ensure new content matches existing style/tone
7. Validate entire file still passes validation
8. Regenerate .md from updated .txt
```
